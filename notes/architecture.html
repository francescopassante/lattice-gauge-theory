Gauge-Equivariant Graph Attention Network

GELT — Implementation Plan

A gauge-equivariant attention architecture for SU(Nc) lattice gauge theory. Follows the L-CNN framework (arXiv:2012.12901) for primitives and proofs, with two departures:

  1. The L-Conv + L-Bilin stack is replaced by an attention block whose value path is matrix-bilinear (not scalar-weighted), so L-CNN's loop-doubling universality argument transfers directly.
  2. Parallel transport is averaged over all shortest lattice paths rather than fixed to one axis-aligned path, so each block already reaches the full L1-ball receptive field with non-axis-aligned loop content.

Equation references like "Eq. 5" point to gauge_invariant_NN_review.md in this directory.

1. Conventions and data structures

1.1 Lattice

  • Hypercubic lattice Λ of shape (N_t, N_s, …, N_s), dimension D+1, periodic on every axis.
  • Sites x ∈ Λ; direction index μ ∈ {0,…,D}.
  • N_c colors (use N_c = 2 first).

1.2 Tensors

Everything is complex (torch.complex64/128). If complex autograd is incomplete, split (Re, Im) and process as 2·N_c² real channels.

  • Links U: (B, D+1, *Λ, N_c, N_c), U[b,μ,x] ∈ SU(N_c).
  • Covariant field W: (B, C, *Λ, N_c, N_c), W[b,i,x] ∈ ℂ^{N_c × N_c}.
  • Inverse links: U_{x+μ̂,-μ} = U†_{x,μ}, computed on demand.

1.3 Gauge transformation laws

  • Links: U_{x,μ} → Ω_x U_{x,μ} Ω†_{x+μ̂}.
  • W-channels: W_{x,i} → Ω_x W_{x,i} Ω†_x.

A function is equivariant if f(T_Ω U, T_Ω W) = T_Ω f(U, W), invariant if f(T_Ω U, T_Ω W) = f(U, W). Every layer is one or the other by construction — never trained.

2. Preprocessing (fixed, non-trainable)

2.1 Plaq

U_{x,μν} = U_{x,μ} U_{x+μ̂,ν} U†_{x+ν̂,μ} U†_{x,ν}    for μ < ν

Stored as W-channels; output has (D+1)·D/2 channels per site.

2.2 Poly (optional)

Straight Polyakov loop along each axis through each site (Eq. 11). Adds (D+1) channels. Required if non-contractible loops are in the target.

2.3 Channel augmentation (augment)

Before any bilinear, expand C channels to (2C + 1) by prepending the identity and appending Hermitian conjugates:

W → [ 1, W_1, …, W_C, W†_1, …, W†_C ]

Gives bias and orientation-reversal for free; the only way the bilinear block can reduce to identity at init. Apply inside the G-Attn block before Q, K, V.

2.4 Trace (invariant readout, Eq. 10)

T_{x,i} = Tr[ W_{x,i} ]

Used only at the end. Feed Re T and/or Im T into a small MLP head.

3. The G-Attn block

One block maps W^{(in)} → W^{(out)} with the same shape, gauge-equivariantly, and bakes a matrix product into the output, doubling the loop length per block (same scaling as L-CB: n_blocks ≥ ⌈log₂ N²⌉ for an N×N loop).

3.1 Hyperparameters

SymbolMeaning
Hnumber of attention heads
dper-head channel dim (total = H·d)
Rattention range — Manhattan radius of the L1-ball
offsetsΔx ∈ ℤ^{D+1}, 0 < |Δx|_1 ≤ R

Offset count in D+1 = 4 (generator [(1+z)/(1−z)]^4): R=1 → 8, R=2 → 40, R=3 → 128, R=4 → 320.

3.2 On-site channel projections (Q, K, V)

Augment W^{(in)} with 1 and conjugates → W̃, C̃ = 2C + 1. For each head h, three independent complex weights w^Q_h, w^K_h, w^V_h ∈ ℂ^{d × C̃}:

Q_{x,h,a} = Σ_j w^Q_{h,a,j} · W̃_{x,j}      a ∈ {1..d}
K_{x,h,a} = Σ_j w^K_{h,a,j} · W̃_{x,j}
V_{x,h,a} = Σ_j w^V_{h,a,j} · W̃_{x,j}

Each is a stack of H·d matrices in ℂ^{N_c × N_c}, locally covariant: Q_{x,h,a} → Ω_x Q_{x,h,a} Ω†_x.

Implementation: one einsum("hajc,bjcx…mn->bhax…mn", w, W̃) per Q/K/V.

3.3 Parallel transport: average over shortest paths

For neighbor y = x + Δx, a single axis-aligned path is gauge-covariant but arbitrary. The unweighted sum over all shortest lattice paths is also gauge-covariant (sum of covariants), uses no direction preferentially, and is strictly more expressive per block:

T_Δx(x)  =  Σ_{P : x → x+Δx, |P| = |Δx|_1}  U_P      (gauge-covariant)

By linearity, T_Δx(x) → Ω_x · T_Δx(x) · Ω†_{x+Δx}.

The unweighted sum keeps the multinomial path multiplicity (e.g. (2,2,0,0) has six paths, (4,0,0,0) has one), introducing an offset-dependent scale asymmetry. Normalized transports (§12) are a cheap ablation once the baseline passes §9.1.

Compute via DP, not enumeration. Recurse on the first step (sign of Δx_μ selects forward or backward link):

T_Δx(x)  =  Σ_{μ : Δx_μ > 0}  U_μ(x)        · T_{Δx − ê_μ}(x + ê_μ)
          + Σ_{μ : Δx_μ < 0}  U†_μ(x − ê_μ) · T_{Δx + ê_μ}(x − ê_μ)

Build T_Δx for every signed Δx in the L1-ball of radius R, in order of increasing |Δx|_1. Cost per new offset: ≤ D+1 matmuls per site, independent of path count.

Octant identity. T_{−Δx}(x) = dagger(T_Δx(x − Δx)) holds as a math property (sum reverses under dagger-and-shift) and is used as a test-suite check. We store the full signed L1-ball anyway: at R ≤ 3 in 4D the table fits comfortably, and one auditable surface is worth more than the 2× memory saving — mixed-sign offsets like (1, −1, 0, 0) cannot be derived from positive-octant data in any case.

Transport K and V back to x:

K̃_{Δx→x,h,a} = T_Δx(x) · K_{x+Δx,h,a} · T†_Δx(x)
Ṽ_{Δx→x,h,a} = T_Δx(x) · V_{x+Δx,h,a} · T†_Δx(x)

Both are locally covariant at x. Build T_Δx once per batch and reuse across H heads and the K, V branches; this is the dominant 4D cost and benefits substantially from torch.compile.

4D memory for the full signed L1-ball (complex64, N_c = 2, shared by K and V):

LatticeR=2 (40)R=3 (128)R=4 (320)
4⁴<1 MB<1 MB~5 MB
8⁴~10 MB~30 MB~80 MB
16⁴~170 MB~540 MB~1.3 GB

Multiply by batch size. R = 3 is the practical sweet spot up to 16⁴; R = 4 is feasible at ≤ 8⁴. Don't exceed R = 4 — the loop-doubling rule cheaply outgrows any single-block reach.

3.4 Attention score (gauge invariant)

Per head, per neighbor:

s_{x→y,h} = (1 / √(N_c · d)) · Re Tr [ Σ_a Q†_{x,h,a} · K̃_{y→x,h,a} ]

Both factors transform as Ω_x · (·) · Ω†_x at x; Re Tr is invariant by cyclicity. The Σ_a plays the role of the per-head feature sum in standard attention. Tr[Q† · K̃] is the Frobenius inner product of Q and K̃ — the natural matrix generalisation of q† k.

Relative position bias. Translation equivariance is automatic; per-offset learned scalars are still useful:

s_{x→y,h}  +=  b_h(Δx)        (real, trainable)

Tie b_h across the lattice point-group's orbit of Δx (90° rotations + parity) by default for isotropic targets (Wilson loops, action, topological charge). Untie only when the physics distinguishes axes (e.g. finite temperature).

Identity at init. The ReZero scalar α (§3.8) is zero-initialized, so the block returns W^{(in)} bit-exactly at init regardless of what the value path / score / gate produce. Small w^V, w^{mix} still matter as a warm-start for the post-ReZero regime (they keep the gradient through α on a sane scale once α ramps up), and the score being approximately uniform at init means the early attention pattern is isotropic. No score-reference subtraction is needed.

Physical interpretation. Substituting the transport rule into the score gives, schematically:

s_{x→y,h,a} ∝ Re Tr [ Q†_{x,h,a} · T_Δx(x) · K_{y,h,a} · T†_Δx(x) ]

This is exactly the gauge-invariant two-loop correlator of lattice physics — the shape of glueball propagators, string-tension correlators, and Polyakov-loop correlators. The block asks, at every site and neighbor, "how correlated is my local loop content with my neighbor's, after parallel-transporting it into my frame?" High score when Q_x and transported K_y are color-aligned (correlated gauge content); low when uncorrelated; approximately uniform at init. CASK (2501.16955) demonstrates the same Frobenius-style score on smeared links for SU(3) pure gauge, so the inductive bias is at least not broken.

3.5 Softmax

For each (x, h), normalize over the neighborhood:

α_{x→y,h} = exp(s_{x→y,h}) / Σ_{y' ∈ N(x)} exp(s_{x→y',h})

Softmax of invariant scalars stays invariant. For additive local targets (action / charge density), tanh(s) without normalization is a peer alternative; loop-doubling does not depend on the choice.

3.6 Multiplicative value path (L-Bilin baked in)

The key departure from a vanilla transformer:

W^{(out)}_{x,h,a}  =  Σ_{y ∈ N(x)}  α_{x→y,h} · Q†_{x,h,a} · Ṽ_{y→x,h,a}

Both Q†_{x,h,a} and Ṽ_{y→x,h,a} are covariant at x, so their product is covariant; the scalar α preserves covariance. The matrix product Q† · Ṽ plays the role of L-Bilin's W · W' — loop length doubles per block.

Reuse of Q† across score and output is intentional: it appears as the left factor in both Tr[Q† · K̃] and Q† · Ṽ, keeping paths consistent and reducing parameter count. Decouple via a fourth projection Q' if needed later (§12).

3.7 Channel mix back to C output channels

W^{(out)} has H·d channels. Mix to C_out via a trainable complex map (no transport, on-site):

W^{(out,mixed)}_{x,i}  =  Σ_{h,a}  m_{i,h,a} · W^{(out)}_{x,h,a}      m ∈ ℂ

3.8 Residual + L-Act (ReZero)

W^{(res)}_{x,i} = W^{(in)}_{x,i} + W^{(out,mixed)}_{x,i}
W^{(act)}_{x,i} = g_{x,i}(U, W^{(res)}) · W^{(res)}_{x,i}
W^{(block)}_{x,i} = W^{(in)}_{x,i} + α · ( W^{(act)}_{x,i} − W^{(in)}_{x,i} )

Multiplication by an invariant scalar preserves covariance. Two viable gates:

g_relu(W)     = ReLU( Re Tr[W] / N_c )         # L-CNN default
g_softplus(W) = softplus( Re Tr[W] / N_c )      # never vanishes

Caveat for SU(N), N > 1. Tr[W] projects out the traceless part; products of plaquettes can be approximately traceless after a few blocks, in which case ReLU kills the channel. Prefer g_softplus beyond 2–3 blocks, or use ReLU(Re Tr[W]/N_c + b₀) with small positive b₀.

Why ReZero and not just residual + L-Act. A naïve residual W^{(act)} = g(W^{(res)}) · W^{(res)} is not identity at init: with small w^V, w^{mix} we have W^{(res)} ≈ W^{(in)}, but the gate then rescales by g(W^{(in)}). For g_softplus and the natural input regime (Re Tr[W]/N_c ≈ 0 on Haar-random or near-Haar configs), g ≈ softplus(0) = ln 2 ≈ 0.693, so each block multiplies W by ~0.7 — an exponential decay with depth that kills the deep-stack stability the residual was supposed to give. Empirically the L-Frobenius drift ‖attn(W,T) − W‖ / ‖W‖ is ≈ 0.56 after two GEMHSA blocks on SU(2) Haar plaquettes with g_softplus, far from identity.

The fix is the standard ReZero / LayerScale trick: introduce one learnable real scalar α per block, zero-initialized, and blend W^{(block)} = W^{(in)} + α · (W^{(act)} − W^{(in)}). At init α = 0 so the block is bit-exactly the identity, regardless of gate choice or input scale; during training α grows and the gate/mix path takes over. α is gauge-invariant by construction (a real scalar), so the convex combination of two equivariant terms remains equivariant — the §7 stress test is unaffected. This is what makes the §5 "identity at init" property hold literally, not merely up to a global multiplicative factor, and is what enables stable deep stacking.

3.9 Pseudocode

def G_Attn_block(U, W_in, weights):
    # 3.2 augment + project
    W_aug = augment(W_in)                                  # (B, 2C+1, *Λ, Nc, Nc)
    Q = einsum("haj,bjxmn->bhaxmn", weights.wQ, W_aug)
    K = einsum("haj,bjxmn->bhaxmn", weights.wK, W_aug)
    V = einsum("haj,bjxmn->bhaxmn", weights.wV, W_aug)

    # 3.3 shortest-path-averaged transports for every signed Δx in L1-ball
    T = build_transport_sums(U, R)            # dict: Δx -> (B, *Λ, Nc, Nc)
    scores, Vt = [], []
    for dx in offsets_in_L1_ball(D+1, R):
        T_dx    = T[dx]
        K_shift = roll_multi(K, shift=tuple(-d for d in dx))  # bring K_{x+Δx} to index x
        V_shift = roll_multi(V, shift=tuple(-d for d in dx))
        Kt      = T_dx @ K_shift @ dagger(T_dx)
        Vt_off  = T_dx @ V_shift @ dagger(T_dx)
        # 3.4 score
        prod = einsum("bhaxnm,bhaxnp->bhaxmp", conj(Q), Kt)
        s    = real(trace(prod, axes=(-2,-1))).sum(axis_a)    # (B, H, *Λ)
        s   /= sqrt(Nc * d)
        s   += weights.bias[orbit_of(dx)]
        scores.append(s); Vt.append(Vt_off)

    # 3.5 softmax over offsets
    alpha = softmax(stack(scores, axis=offset), axis=offset)  # (B, H, *Λ, |offsets|)

    # 3.6 multiplicative value
    W_out_hd = zeros_like(Q)
    for o, dx in enumerate(offsets_in_L1_ball(D+1, R)):
        W_out_hd += alpha[..., o, None, None] * (dagger(Q) @ Vt[o])

    # 3.7 channel mix
    W_mix = einsum("iha,bhaxmn->bixmn", weights.mix, W_out_hd)

    # 3.8 residual + L-Act + ReZero
    W_res = W_in + W_mix
    gate  = softplus(real(trace(W_res, axes=(-2,-1))) / Nc)
    W_act = gate[..., None, None] * W_res
    return W_in + weights.alpha * (W_act - W_in)   # alpha: real scalar, init 0

3.10 Scope: gauge but not point-group equivariance

Every layer is exactly gauge-equivariant. The block as a whole is not exactly point-group equivariant, even with orbit-tied biases (§3.4). The biases make the offset attention pattern an isotropic prior; they cannot make Q/K/V projections commute with lattice rotations, because the input embedding itself is not pointwise rotation-covariant.

Why. §2.1 anchors P_{μν}(x) at one corner (μ < ν, lowest-index). Under a 90° rotation R, the loop rotates rigidly but its anchor does not map to R·x; the plaquette field transforms as (R·P)_{μν}(x) = ±P_{σ(μν)}(R⁻¹x + c_{μν}) with a channel-dependent shift c_{μν}. No single translation absorbs c uniformly, so neither weight sharing nor orbit-tied weights can rescue point-group equivariance.

The fix lives at the input embedding (§12 all-leaves variant), not in the block. Until adopted, treat point-group symmetry as a bias prior, not a guarantee — §7-style tests should not be expected to certify rotation invariance.

4. Full network

U  ──▶  Plaq, (Poly?)  ──▶  W^(0)  ──▶  G_Attn_block × L  ──▶  W^(L)
                                                                  │
                                                                  ▼
                                                Trace + (Re, Im)  ──▶  small MLP (per-site)
                                                            (NO global average pool)

4.1 Choosing L

L = ⌈log₂(N²)⌉ for an N×N Wilson loop. For 3+1D topological charge density: L = 4–6, H = 2, d = 4, R = 2, plaquette + rectangle channels in W^(0).

4.2 No global average pool

Emit per-site predictions; aggregate after the per-site head. Per L-CNN supp. Table XVI, replacing per-site with GAP-then-MLP destabilizes training even when the target is itself global.

Aggregation must match the observable's normalization:

Action Saction densitysum
Avg plaquette ⟨P⟩local plaquettemean
Topological charge Qcharge density q(x)sum
Fixed Wilson looploop rooted at xmean

4.3 Optional L-Exp head

For tasks updating links (HMC, normalizing flow, gradient flow), append:

U_{x,μ} → exp( i Σ_i β_{μ,i} · [W^(L)_{x,i}]_ah ) · U_{x,μ}

with [X]_ah = (X − X†)/(2i) − Tr(X − X†)/(2i·N_c) · 1. Re-run Plaq/Poly after each L-Exp.

5. Initialization

  • Complex projections w^Q, w^K, w^V and channel mix m: small Gaussian, σ ≈ 0.02 / √C̃. Real and imaginary parts independent.
  • Position biases b_h(Δx) (one per orbit): zero.
  • L-Exp β: zero.
  • ReZero scalar α (§3.8): zero. This is what makes the identity property hold bit-exactly; small-σ projections alone are insufficient because the gate g_softplus(0) = ln 2 ≠ 1 rescales W on every forward.
  • With α = 0 each block returns W^{(in)} verbatim, so the network is equivalent to W^{(0)} piped straight to Trace. Attention pattern is irrelevant at init (any α-scaled artifact is zeroed out). Verify empirically before training (drift ‖attn(W,T) − W‖ / ‖W‖ should be at machine ε).

6. Training

6.1 Loss

  • Regression: MSE.
  • Tiny labels (topological charge density): pre-multiply labels by C_scale (e.g. 100, per L-CNN); divide back at inference.

6.2 Optimizer / schedule

  • AdamW, weight decay 0.
  • LR: 3e-3 small Wilson loops, 1e-3 medium, 3e-4 topological charge.
  • Batch size 50; 20 (small) / 100 (large) epochs; early stopping patience 5–25.
  • Train each architecture with ≥ 5 random seeds; report median.

6.3 Sanity checks during training

  • Loss must be identical to machine precision under random Ω_x (single: ~1e-6; double: ~1e-14).
  • At init, prediction must equal the Trace-of-W^(0) baseline (residual identity).

6.4 Parameter counting

For matched-parameter comparison against real-valued baselines, count each complex weight as two real parameters. Projections w^Q, w^K, w^V ∈ ℂ^{d × C̃} and mix m ∈ ℂ^{C_out × H × d} are complex; not doubling them silently gives GELT 2× capacity. Biases b_h(Δx) ∈ ℝ count as one per point-group orbit; untying inflates by the average orbit size (8–24 in 4D).

7. Validation: gauge invariance tests

Required for any thesis-level claim of equivariance.

7.1 Random gauge transformation

Sample χ^a_x ~ N(0, α²) per generator a and site x; form Ω_x = exp(i t^a χ^a_x). Apply to (U, W). Measure relative drift:

|y_pred(U, W) − y_pred(T_Ω U, T_Ω W)| / |y_pred(U, W)|

Target: < 1e-5 single, < 1e-13 double.

7.2 Gauge-implementation stress test

A unit test of the implementation: if exactly equivariant, loss is analytically zero for all Ω; the optimizer only probes the float-point ceiling. Finds worst-case Ω that exposes implementation bugs hidden in a thin slice of gauge space.

Parametrize Ω_x = exp(i t^a ρ^a_x) with trainable ρ^a_x ∈ ℝ. Optimize ρ via AdamW (lr 1e-2, ~200 steps) to maximize

L_stress = | y_pred(U, W) − y_pred(T_Ω(ρ) U, T_Ω(ρ) W) |

A correctly-implemented GELT keeps L_stress bounded by floating-point noise. Anything larger is a bug — almost always a missed dagger or a non-axis-aligned transport path.

8. Universality argument

Each block produces, at x, matrices of the form α(invariant) · Q†_x · U_P · K^or^V_y · U†_P — a parallel-transported, attention-weighted bilinear. By the same induction as L-CNN's Sec. 3.2:

  • One block with unit kernels gives all 1×k loops up to k = R, attention-weighted.
  • Stacking L blocks doubles max loop length per block, reaching 2^L.
  • L-Act adds non-linearity in the trace channel.
  • With Plaq + Poly preprocessing, the network represents any non-linear function of any Wilson/Polyakov loop in its receptive field.

Because transport is averaged over all shortest paths in the L1-ball (§3.3), a single block already produces every shortest-path loop closing within |Δx|_1 ≤ R — including multi-axis and "corner" loops. This recovers per-block the non-axis-aligned loop content that L-CNN's axis-aligned L-Conv only reaches by stacking, and gives a strictly richer per-block shape set than CASK's s = 1..R extended staples (which fix one ν per staple). Loop-doubling per block still holds via Q† · Ṽ.

9. Reference test problems

Replicate L-CNN's benchmarks first to confirm soundness.

9.1 Wilson loop regression (1+1D, SU(2))

  • Lattice 8 × 8, β ∈ {0.1, …, 6.0}, 10⁴ train / 10³ val / 10³ test.
  • Target: (1/N_c) Re Tr[U^{(m×n)}] for (m,n) ∈ {(1,2), (2,2), (4,4)}.
  • Test generalization on 16², 32², 64² without retraining.
  • Pass: MSE within 1× of L-CNN at matched params (L-CNN: 1e-8–1e-7); beat baseline CNN by ≥ 4 orders.

9.2 Topological charge density (3+1D, SU(2))

  • Lattice 4 × 8³ train, up to 8 × 16³ test.
  • Target: plaquette discretization (Eq. 13).
  • Prefer flowed/cooled labels — raw q(x) is noisy and the model will learn discretization artifacts.
  • Combine with Wilson flow at inference; recover near-integer Q_P.

9.3 Trained-model stress test

Run §7.2 on the trained W^(1×2) model. Target: drift < 1e-5 (single) / < 1e-13 (double).

10. Build order

Build in this order. Verify each step before moving on.

  1. Lattice utilities: roll-with-PBC, dagger, build_transport_sums(U, R) — DP for shortest-path-averaged transports over the full signed L1-ball (§3.3), ordered by |Δx|_1. Verify vs. brute force for |Δx|_1 ≤ 2 (40 offsets in 4D); test the octant identity under a random Ω.
  2. Plaq, Poly, augment, Trace (fixed). Apply random Ω; check transformation laws to machine precision.
  3. On-site Q/K/V. Test Q(T_Ω W̃) = T_Ω Q(W̃).
  4. Parallel transport: covariance unit test.
  5. Score Re Tr[Q† · K̃]: gauge-invariance unit test.
  6. Softmax: numerically stable (subtract max).
  7. Value path α · Q† · Ṽ: covariance unit test.
  8. Channel mix + residual + L-Act: full-block covariance unit test.
  9. Trace + MLP head: full-network invariance unit test.
  10. §7.1 + §7.2 on the untrained model — must pass before training.
  11. Train on W^(1×2) regression; reproduce L-CNN-level MSE.
  12. Scale up.

11. PyTorch-specific notes

  • Use torch.einsum for all contractions — gauge equivariance becomes manifest in code.
  • If complex autograd is a problem, split tensors with a final dim of size 2 (real, imag) and provide custom matmul/dagger; benchmark first — recent PyTorch supports complex autograd for most ops.
  • Cache transport products U_P per batch; reused across heads and K/V branches.
  • Use torch.compile on the per-block forward; the einsum kernel benefits substantially.
  • Train in float32; validate gauge invariance in float64 (cast model + one batch).

12. Variants (after baseline works)

  • Decoupled output query Q': separate from score Q. +33% projection params; useful when score and bilinear want different feature mixes.
  • CASK-style extended staples: replace L1-ball transport by 1×s staples (one ν per staple). Strictly less expressive per block; useful as a CASK-baseline ablation.
  • Single fixed shortest path (L-CNN style): one chosen path per offset. Cheaper but breaks the discrete point-group symmetry of transport. Ablation to isolate path-averaging.
  • Path-count-normalized transports: T_Δx = (1/n_paths) Σ_P U_P, with n_paths = |Δx|_1! / Π_μ |Δx_μ|!. Removes the mixed-axis norm advantage while preserving covariance.
  • Tanh score instead of softmax: more natural for additive local observables.
  • Richer L-Act gate: small MLP over local invariants (Re Tr W_i, Im Tr W_i, Re Tr(W_i W_j)). Stays scalar/invariant; helps deep stacks where Tr W_i collapses.
  • Pre-norm two-residual stream: separate attention residual from L-Act residual.
  • Edge-feature attention: condition b_h(Δx) on local plaquette invariants too.
  • Untied biases: anisotropy ablation, or for targets where lattice axes are physically inequivalent.
  • Full point-group equivariance via all-leaves input. The anchored-plaquette embedding (§2.1) is the structural reason the block is gauge- but not point-group-equivariant (§3.10). At each site x and pair (μ, ν), the four 1×1 plaquettes touching corner x carry the regular representation of the local 4-fold rotation: under 90° they permute cyclically and stay attached to x. Store all four as separate channels of W^(0), each traversed starting and ending at x. Each leaf must be rebuilt as a new ordered link product; you cannot reuse the neighbor's anchored P_{μν}(x − ê_μ) (covariant at x − ê_μ, not at x). Under rotation, (R·Q^(i))(x) = Q^(σ(i))(R⁻¹x): pointwise channel permutation, no residual shift. Combined with orbit-tied linear maps (every trainable map commutes with σ) and the already-equivariant transport, the block becomes exactly hypercubic-equivariant. Cost: 4× input channels per pair (4D: 6 → 24). Symmetrization is needed only on the input; deeper layers inherit equivariance from orbit-tying. Verify via §7.2 analogue: rotate U, permute channels by σ, check drift at machine ε. Higher D: orbit-tie across joint (pair, leaf) tuples since hypercubic permutes pair labels too. Polyakov loops have no leaf decomposition; tie across axis-permutation orbits. Clover variant: C_{μν}(x) = Σ_i Q^(i)(x) is 4× cheaper but projects onto the trivial irrep, discarding the three non-trivial irreps; the standard improved q(x) discretization for that reason. Use the full leaf set unless memory rules it out — orbit-tied weights can learn the symmetric sum on their own.
